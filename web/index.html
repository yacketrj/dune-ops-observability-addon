KPI Capability

Read-only support map

Shows which reporting categories are actually live right now, computed from each data source's real status on the most recent refresh — never a static claim. A capability backed by more than one source (e.g. Population & Activity) shows "partial" if only some of its sources are live.

Population & Activity

—

Player totals, online status, active windows, guild/faction/map distribution via ops.health.* and ops.activity.summary.

Farm Health

—

Farm totals, ready count, and alive count are available from ops.health.farms.

Combat & Deaths

—

PvP/PvE death breakdowns, cause, map, and NPC tracking via ops.combat.deaths.

Resources

—

Resource field inventory, value remaining, type and map distribution via ops.resources.summary.

Economy & Trade

—

Currency supply, exchange orders, fulfilled trades, and tax fees via ops.economy.summary.

Inventory & Crafting

—

Item counts by template, inventory usage, and crafting totals via ops.inventory.summary.

SOC

—

Bridge-request health and error-rate summary via ops.soc.summary.

Metrics (Prometheus)

—

Optional Prometheus/cAdvisor/node-exporter stack health via ops.health.prometheus. Requires the operator to run `dune metrics start` — "unavailable" here commonly means the stack isn't running, not a defect.
