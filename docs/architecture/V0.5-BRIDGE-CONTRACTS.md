## ops.economy.summary (new — this release)

```
POST /api/addons/installed/{id}/bridge
Body: { "action": "ops.economy.summary" }
Permission: ops:read
```

### Response Schema

```json
{
  "ok": true,
  "result": {
    "totalCurrencyHolders": 12,
    "totalSupply": 250000,
    "activeOrders": 5,
    "fulfilledOrders": 47,
    "taxCollected": 3250,
    "currencyBreakdown": [
      {
        "currencyId": "0",
        "holders": 12,
        "supply": 250000,
        "avgBalance": 20833,
        "minBalance": 50,
        "maxBalance": 150000
      }
    ],
    "topTradedItems": [
      {
        "templateId": "ScrapMetal",
        "orders": 15,
        "avgPrice": 50,
        "minPrice": 10,
        "maxPrice": 200
      },
      {
        "templateId": "HeavyAmmo",
        "orders": 8,
        "avgPrice": 120,
        "minPrice": 80,
        "maxPrice": 200
      }
    ]
  }
}
```

### Empty State

```json
{
  "ok": true,
  "result": {
    "totalCurrencyHolders": 0,
    "totalSupply": 0,
    "activeOrders": 0,
    "fulfilledOrders": 0,
    "taxCollected": 0,
    "currencyBreakdown": [],
    "topTradedItems": []
  }
}
```

### Failure Mode

When any economy table is missing or a query fails:
- Returns empty/default values for that section
- Does NOT throw — all queries wrapped in `try/catch`
- Always returns `{ ok: true, result }` with zero-filled object

### Addon Consumer

```js
// renderEconomy(data) in addon.js expects:
ecoHoldersEl  ← data.totalCurrencyHolders
ecoSupplyEl   ← data.totalSupply
ecoOrdersEl   ← data.activeOrders
ecoFulfilledEl ← data.fulfilledOrders
ecoTaxEl      ← data.taxCollected
ecoCurrencyBodyEl ← data.currencyBreakdown (table rows)
ecoTradeBodyEl    ← data.topTradedItems (table rows)
```
