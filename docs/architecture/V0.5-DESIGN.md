# v0.5.0 — Economy & Resources Design

**Release**: v0.5.0
**Status**: Implemented — `ops.economy.summary` is live in Core (`console/api/src/server.js`, `duneDb.addonOpsEconomySummary()`) and called by the addon's Economy panel (`web/data-providers.js`'s `getEconomy()`). Confirmed via `docs/SECURITY-ARCHITECTURE-GAP-ANALYSIS.md` (2026-07-22).
**Date**: 2026-07-07
**Depends on**: Core PR #68 (ops.activity/resource/combat bridge actions)

---

## Abstract

Adds `ops.economy.summary` bridge action to Core, enabling the Economy tab in the addon. The `ops.resources.summary` action (spice field tracking) is already implemented in PR #68. This release completes the remaining half: currency supply, exchange orders, and tax tracking.

---

## Component Specification

### ops.resources.summary (implemented — PR #68)

| Field | Type | Source | Description |
|---|---|---|---|
| `totalFields` | int | `resourcefield_state` WHERE field_kind_id=1 | Total active spice fields |
| `totalValueRemaining` | bigint | `SUM(value_remaining)` | Total harvestable spice |
| `resourcesByMap` | array | GROUP BY map | Field count + value per map |
| `spiceFieldsBySize` | array | `spicefield_types` JOIN | Size breakdown (Small/Medium/Large) with active/cap |

Already deployed on e2e. Render function: `renderResources()` in addon.

### ops.economy.summary (new — this release)

| Field | Type | Source | Description |
|---|---|---|---|
| `totalCurrencyHolders` | int | `player_virtual_currency_balances` | Distinct players with currency |
| `totalSupply` | bigint | `SUM(balance)` | Total currency in circulation |
| `activeOrders` | int | `dune_exchange_orders` | Currently active exchange orders |
| `fulfilledOrders` | int | `dune_exchange_fulfilled_orders` | Completed exchange orders |
| `taxCollected` | bigint | `tax_invoice` | Total tax/fees collected |
| `currencyBreakdown` | array | GROUP BY currency_id | Per-currency holders, supply, avg/min/max |
| `topTradedItems` | array | GROUP BY template_id | Orders, avg price, min/max per item |

All exchange queries are defensive — wrapped in `tableExists()` + `try/catch`. Tables may have 0 rows on fresh servers.

## Security

| Concern | Mitigation |
|---|---|
| Currency data exposure | Aggregate only — no per-player balances exposed |
| Exchange data exposure | Aggregate order counts + price stats — no individual orders |
| Tax data exposure | Total tax collected — no per-player tax records |
| SQL injection | All queries parameterized via `db.query(text, values)` |
| Permission | `ops:read` via `assertInstalledAddonPermission()` |
