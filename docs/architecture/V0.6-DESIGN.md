## Abstract

Adds `ops.inventory.summary` and `ops.location.activity` bridge actions. Enables the Inventory and Location tabs in the addon. Both require database discovery re-run against production data to confirm table schemas and row counts.

---

## ops.inventory.summary

| Field | Type | Source | Description |
|---|---|---|---|
| `totalItems` | int | `items` | Total items across all inventories |
| `totalInventories` | int | `inventories` | Distinct inventory containers |
| `totalCrafted` | int | `item_operations_staging_table` WHERE function_name = 'save_item' | Items crafted (may be 0 rows) |
| `itemsByTemplate` | array | GROUP BY template_id | Count + stack size per item template |
| `storageByContainer` | array | GROUP BY inventory_id | Items + total stack per container |

## ops.location.activity

| Field | Type | Source | Description |
|---|---|---|---|
| `activeMaps` | int | `map_names` DISTINCT | Maps with activity |
| `totalMarkers` | int | `markers` | Total map markers |
| `playerDensityByMap` | array | `overmap_players` GROUP BY map_name_id | Players + online per map |
| `markersByMap` | array | `markers` GROUP BY map_name_id | Marker count per map |

## Security

- Inventory data is sensitive — aggregate only by template, no per-player items
- Location data is coarse (map/zone level) — no coordinates exposed
- Both use defensive `columnsFor()` + `tableExists()` pattern
